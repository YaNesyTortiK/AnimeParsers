# Токены для кодика

В данной библиотеке для парсера кодика используются токены которые находятся в открытом доступе либо в нем находились.
Токены хранятся в файле `kdk_tokns/tokens.json` и при каждой инициализации класса парсера выполняется запрос к данному файлу для получения списка токенов и последующей их проверки.
Поэтому рекомендуется после первого запроса, после которого был найден рабочий токен, получить его из парсера примерно таким образом:
```python
parser = KodikParser()
token = parser.TOKEN
```
После чего этот токен следует указывать в параметрах парсера, чтобы избежать задержки на получение списка токенов и проверки.


## Шифрование токенов
Токены специально зашифрованы, чтобы не было возможности найти эти токены через запрос в поисковых системах (избегание индексации)

Для зашифровывания используется следующая функция:
```python
from base64 import b64encode

def encrypt_token(token: str) -> str:
    p1 = b64encode(token[:16].encode('utf-8')).decode('utf-8')
    p2 = b64encode(token[16:].encode('utf-8')).decode('utf-8')
    return p2[::-1]+p1[::-1]
```
То есть берется первая половина токена, кодируется в base64. Затем берется вторая половина и также кодируется в base64. Затем перевернутая вторая часть складывается с перевернутой первой частью.

Функция расшифровки (выполняет операции в обратном порядке):
```python
from base64 import b64decode

def decrypt_token(tkn: str) -> str:
    p1 = tkn[:len(tkn)//2][::-1]
    p2 = tkn[len(tkn)//2:][::-1]
    p1 = b64decode(p1.encode('utf-8'))
    p1 = p1.decode('utf-8')
    p2 = b64decode(p2.encode('utf-8'))
    p2 = p2.decode('utf-8')
    return p2+p1
```

## Схема файла
Файл `tokens.json` разделен на несколько разделов:
* stable - полностью рабочие токены которые, по наблюдениям, меняются реже всего
* unstable - полностью рабочие токены которые, по наблюдениям, меняются чаще чем stable токены
* legacy - неполноценные токены, которые не работают для всех функций, но работают для функций `get_info`, `get_link`, `get_m3u8_playlist_link` (к этим токенам также относится токен который находится в одном из файлов кодика)
* dead - токены которые не работают ни для одной функции. Они записаны чтобы была возможность отслеживать какие токены уже были выведены из оборота

Каждый раздел содержит токен, результат тестирования доступа для основных функций, временная метка последней проверки.

Примерная структура:
```json
{
    "stable": [
        {
            "tokn": "Зашифрованный токен",
            "functions_availability": {
                "base_search": true,
                "base_search_by_id": true,
                "get_list": true,
                "search": true,
                "search_by_id": true,
                "get_info": true,
                "get_link": true,
                "get_m3u8_playlist_link": true
            },
            "last_checked": "YYYY-mm-dd HH:MM:SS.ms"
        }
    ],
    "unstable": [
        
    ],
    "legacy": [

    ],
    "dead": [
        
    ]
}
```
